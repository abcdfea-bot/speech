**接口设计**

**外部接口（用户界面 & 软件接口）**
**上传接口（HTTP）:**  
  **路径/方法**: `POST /predict/`  
  **请求头**: `Content-Type: multipart/form-data`  
  **请求体**: 多部分文件字段 `files`（允许批量上传）。每个文件原始格式任意（非 WAV 将被转换）。  
  **成功响应 200**: JSON 数组，每项结构：  
    **filename**: 文件名（string）  
    **result_label**: `"真实"` 或 `"伪造"` 或 `"错误"`  
    **score**: 真实分数（float）取自模型输出 `batch_out[0,1]`  
    **is_bonafide**: 布尔（score > threshold）  
    **error**: 可选，错误信息（string 或 null）  
  **错误响应示例**: 400/415/500；body 可包含 `{ "detail": "错误描述" }`。  
  **行为说明**: 并行/批量上传时逐个文件处理并返回结果数组。转换（非 WAV→WAV）由服务器端使用 `pydub` + 本地 ffmpeg.exe 进行。

**前端页面（UI）:**  
  文件: index.html 提供的简单上传页面。  
  功能: 选择多个音频文件 → 显示上传进度 → 请求 `POST /predict/` → 展示每个文件的 `result_label`、`score`、错误信息。  
  推荐交互元素: 文件选择器、多文件预览、上传按钮、结果表格（列：文件名、分数、判定、详细错误/下载原始日志）。

**健康/诊断接口（建议）:**  
  `GET /health` 返回服务就绪状态（模型是否加载、可用 GPU/CPU 信息）。（当前实现未包含，建议添加）

**运行/部署注意:**  
  `uvicorn` 承载 app.py；需确保 ffmpeg.exe 在仓库或系统 PATH 中以支持格式转换。  
  配置文件：config_standalone_eval.json 与权重 epoch_45_0.441.pth 必须可访问。

**内部接口（模块之间）**
**app.py ↔ `predictor.AASISTPredictor`（实例化与调用）**  
  构造器（在 app.py 中）:  
    `AASISTPredictor(model_path: str, config_path: str, threshold: float = 0.5)`  
    作用：加载 `config_path`（含 `model_config`）、构建 `models.AASIST.Model`、加载 `model_path` 权重、设置设备（CPU/GPU）、设定 `cut` 与 `target_sr`、保存 `threshold`。  
  预测方法:  
    `AASISTPredictor.predict(file_path: str) -> dict`  
    输入：音频文件磁盘路径（WAV 已转换或原始 WAV）。  
    输出字典示例：`{"label": "真实", "score": 1.2345, "threshold": 1.510585}` 或 `{"error": "xxx"}`。  
  语义约定：调用方负责把上传文件保存到磁盘并（如需要）先行转换为 WAV，或让 app.py 做转换，然后传 `file_path`。

**app.py ↔ `pydub.AudioSegment`（格式转换）**  
  `AudioSegment.from_file(path)` -> `audio.export(path_wav, format="wav")`  
  输入/输出：任意容器格式文件路径 -> 目标 `wav` 文件路径。  
  约束：需要 `ffmpeg` 可用（本地 ffmpeg.exe 或系统安装）。

**predictor.py ↔ `librosa`（音频加载）**  
  调用: `librosa.load(file_path, sr=self.target_sr, mono=True) -> (X: np.ndarray, sr:int)`  
  约定：返回一维 float numpy 数组，采样率固定为 `self.target_sr`（16000）。

**predictor.py 内部数据契约（预处理到模型）**  
  `pad(x: np.ndarray, max_len: int=64600) -> np.ndarray`：保证长度为 `max_len`；超长截断、过短周期性重复填充。  
  `x_inp = torch.Tensor(X_pad).to(self.device)`，随后 `x_inp = x_inp.unsqueeze(0)` -> tensor shape: `[batch=1, seq_len=64600]`。  
  传递给模型: `models.AASIST.Model.forward(x: Tensor)`，其中 `forward` 期望 `x` 形状为 `[batch, seq]`（函数内部还会 `unsqueeze(1)` 以匹配 conv 通道）。  
  模型返回: `(last_hidden: Tensor, output: Tensor)`；`output` 为 logits，形状 `[batch, 2]`（类别数为2，`batch_out[:,1]` 表示 bonafide 分数）。

**predictor.py ↔ `models.AASIST.Model`（契约总结）**  
  输入：`torch.Tensor` （dtype float32，device 与模型一致）  
  输出：`(Tensor last_hidden, Tensor logits)`  
  error handling：若模型 forward 抛出异常，`predict` 捕获并返回 `{"error": str(e)}`。

**配置接口**  
  config_standalone_eval.json 格式约定：顶层字段 `model_config` 为字典（含 `filts`, `gat_dims`, `first_conv`, `nb_samp` 等）。`AASISTPredictor` 读取并传入 `models.AASIST.Model(model_config)`。

统一错误结构（HTTP JSON）：`{ "error": "短信息", "detail": "可选的长错误堆栈/建议" }`。  
预测失败单条返回仍放在数组中（与成功条目并列），保持一一对应。
