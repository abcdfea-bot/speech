## 问题1、输入张量经过 `conv` 和 `encoder`后的张量形状是什么？每一维的含义是什么？

假设输入批量大小 `(Batch Size)` 为 32，音频长度为 64600。

------

#### 1. `CONV` 层：从 1D 波形到 2D 特征

`CONV` 层是模型的第一步，它是一个 Sinc 卷积层。

**输入张量** (`x`):

- `Model.forward` 接收 `x`，形状为 `(32, 64600)`。
  - `x` 的 `shape` 是 `(batch_size,采样点数)`
  - 采样点数 = 采样率(Hz) × 时间(s)
- `x = x.unsqueeze(1)` 将其变为：`(32, 1, 64600)`
  - `32`: 批量大小 (Batch Size)
  - `1`: 输入通道 (In Channels)
  - `64600`: 原始波形长度 (Length)

**经过 `self.conv_time` (`CONV` 层)**：

- **运算**：`CONV` 层执行 1D 卷积。根据 `CONV` 类的 `forward` 方法，它使用 `F.conv1d`。
- **假设**：`out_channels=70`, `kernel_size=129` 
- **输出长度计算** (Sinc 卷积, stride=1, padding=0)：`L_out = (64600 - 129 + 1) = 64472`。
- **输出张量形状** (`x`)：`(32, 70, 64472)`
  - `32`: 批量大小 (Batch Size)
  - `70`: **Sinc 滤波器数量** (即 `out_channels`，代表不同的频带)
> 这就是之后的 `S`，频域维度
  - `64472`: **特征长度** (即时间维度)
> 这就是之后的 `t`，时间维度

------



## 问题2：encoder里面的结构：

#### 1. `Encoder` 层：从 2D 特征到高级特征图 `e`



在 `CONV` 层之后，`Model.forward` 会继续处理张量，**然后才**将其送入 `encoder`：

**进入 `encoder` 之前的预处理**：

1. `x = x.unsqueeze(dim=1)`
   - *作用*：将 `CONV` 层的输出视为 2D 图像，为 `Residual_block` 中的 2D 卷积做准备。
   - *形状*：`(32, 1, 70, 64472)`
   - *含义*：(批量, 通道=1, **高度=70**, **宽度=64472**)
2. `x = F.max_pool2d(torch.abs(x), (3, 3))`
   - *作用*：2D 最大池化，（默认 stride=3）。
   - *计算H*：`floor((70 - 3)/3 + 1) = 23`
   - *计算W*：`floor((64472 - 3)/3 + 1) = 21490`
   - *形状*：`(32, 1, 23, 21490)`
3. `x = self.first_bn(x)` 和 `x = self.selu(x)`
   - *作用*：归一化和激活。
   - *形状*：`(32, 1, 23, 21490)`

**输入 `encoder` 的张量形状** (`x`)：`(32, 1, 23, 21490)`

------

**经过 `self.encoder`**：

`self.encoder` 是一个 `nn.Sequential`，它包含 6 个 `Residual_block`（**残差块**）。 `Residual_block` 内部的核心形状变化操作是 `self.mp = nn.MaxPool2d((1, 3))`，它（默认 stride=3）会：

- **保持高度 (H) 不变**（`kernel_size=(1, ...)`）。
- **将宽度 (W) 缩减约 1/3**（`kernel_size=(..., 3)`）。

残差块中必然包含至少一个卷积层（通常是 `nn.Conv2d`），其核心参数 `out_channels` 直接决定了输出的通道数。
- 例如：当输入通道数为 in，卷积层的 out_channels 设置为 out 时，输出通道数就会从 in 变为 out。

> 这个通道数就是之后的 `C` 通道维度

下面，我们追踪 `x` 穿过这 6 个块（假设 `filts` 配置 `(1, 32, 32, 64, 64, 64)`）：
1. **Block 1 (in=1, out=32)**：
   - `MaxPool` (1, 3) 作用于 W=21490 -> `floor((21490-3)/3 + 1) = 7163`
   - *形状*：`(32, 32, 23, 7163)`
> - 输入通道数为 1（原始特征，如单通道频谱图）。
> - 块内包含一个卷积层，其 `in_channels=1`，`out_channels=32`。
> 卷积操作通过 **32 个不同的卷积核**（每个核提取一种特征模式）对输入进行卷积，最终输出 32 个通道的特征图。
> - 通道数从 1 扩展到 32。
1. **Block 2 (in=32, out=32)**：
   - `MaxPool` (1, 3) 作用于 W=7163 -> `floor((7163-3)/3 + 1) = 2387`
   - *形状*：`(32, 32, 23, 2387)`
2. **Block 3 (in=32, out=64)**：
   - `MaxPool` (1, 3) 作用于 W=2387 -> `floor((2387-3)/3 + 1) = 795`
   - *形状*：`(32, 64, 23, 795)`
3. **Block 4 (in=64, out=64)**：
   - `MaxPool` (1, 3) 作用于 W=795 -> `floor((795-3)/3 + 1) = 264`
   - *形状*：`(32, 64, 23, 264)`
4. **Block 5 (in=64, out=64)**：
   - `MaxPool` (1, 3) 作用于 W=264 -> `floor((264-3)/3 + 1) = 87`
   - *形状*：`(32, 64, 23, 87)`
5. **Block 6 (in=64, out=64)**：
   - `MaxPool` (1, 3) 作用于 W=87 -> `floor((87-3)/3 + 1) = 28`
   - *形状*：`(32, 64, 23, 28)`

**最终 `encoder` 输出的张量形状** (`e`)：`(32, 64, 23, 28)`

- `32`: 批量大小 (Batch Size)
- `64`: **最终特征通道数** (Feature Channels)
- `23`: **频谱维度 / 高度** (Spectral dimension / Height)
- `28`: **时间维度 / 宽度** (Temporal dimension / Width)
