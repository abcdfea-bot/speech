**仓库关键文件（示例）**  
**入口/服务**: app.py  
**预测封装器**: predictor.py  
**模型实现（PyTorch）**: AASIST.py  
**模型配置**: config_standalone_eval.json  
**前端/测试页面**: index.html

**软件总体结构**  
**部署/接口层（Presentation）**: FastAPI 应用（文件：app.py），负责 HTTP 接口、接收上传文件、返回预测结果与托管前端页面。  
**预处理与文件管理层（IO/Preprocess）**: app.py 中使用临时文件、`pydub` + 本地 `ffmpeg` 做格式转换（非 WAV -> WAV），并清理临时文件。  
**预测服务层（Service / Orchestration）**: `predictor.AASISTPredictor`（文件：predictor.py）负责加载配置和权重、音频加载（`librosa`）、填充/裁剪、把数据转为 tensor 并调用模型推理，封装推理返回的标签与分数。  
**模型实现层（Model）**: `Model`（文件：AASIST.py）为具体的 AASIST 神经网络实现（Graph Attention、SincConv、残差块等），由 PyTorch 运行并返回 logits/score。  
**配置/权重层（Config & Data）**: config_standalone_eval.json 与权重文件（例如 epoch_45_0.441.pth），包含模型结构参数与训练后权重。  
**运行依赖**: `uvicorn`, `fastapi`, `torch`, `librosa`, `pydub`, `ffmpeg` 等（见 requirements.txt）。

**功能模块与程序结构的对应关系**  
HTTP 接收与路由（功能）：实现文件上传批量预测接口。对应程序：app.py 中的 FastAPI 路由 `/predict/` 和根路由 `/`。  
音频格式兼容（功能）：接收任意音频格式并转换为模型接受的 WAV（16kHz）。对应程序：app.py 调用 `pydub.AudioSegment.from_file` + `export`，并设置 `AudioSegment.converter` 指向仓库中的 ffmpeg.exe。  
临时文件管理（功能）：保存上传，转换后清理。对应：app.py 中 `tempfile.NamedTemporaryFile` 与 finally 清理逻辑。  
预处理（功能）：重采样、转单声道、pad/cut 到固定样本长度。对应：predictor.py 中使用 `librosa.load`、`pad` 函数、`self.cut`。  
模型加载与推理（功能）：读取 JSON 配置构造模型、加载 `.pth` 权重、eval 模式推理并提取分数。对应：predictor.py 的 `AASISTPredictor.__init__` 与 `predict()`。实际模型定义在 AASIST.py。  
判定与响应构建（功能）：根据阈值输出“真实/伪造”，构建 JSON 返回。对应：predictor.py 的阈值判断与 app.py 的结果组装。

**模块层次结构与调用关系（自上而下）**  
app.py (FastAPI)  
  调用 -> predictor.AASISTPredictor.predict(file_path)  
    内部调用 -> librosa.load(file_path)（音频加载/重采样）  
    内部调用 -> pad(...)（在 predictor.py 中实现）  
    内部调用 -> models.AASIST.Model (PyTorch) 的 forward(x)  
      使用内部子模块：CONV / Residual_block / GraphAttentionLayer / GraphPool / HtrgGraphAttentionLayer 等（见 AASIST.py）  
    返回 -> logits/score 给 predictor，predictor 计算 label 与 score 字段  
  app.py 收集 predictor 返回，构成 HTTP 响应

调用顺序（单个文件预测）示意：  
Client → POST /predict/ (app.py) → 保存上传 → （非 WAV 则）pydub/ffmpeg 转 WAV → predictor.predict(file) → librosa.load → pad → tensor 化 → Model.forward → output 得分 → predictor 判定 label → app.py 返回 JSON

**模块间边界与职责（简要）**  
app.py：仅负责 HTTP、文件 I/O、格式转换与错误处理（不直接操作模型细节）。保持轻量并作为服务编排器。  
predictor.py：作为服务层与模型层的适配器，负责：读取 `config`, 初始化模型与权重；音频到模型输入的预处理；封装推理输出（异常处理也在此处）。便于未来替换或扩展模型（只需替换 model 类或修改 config）。  
AASIST.py：纯粹的模型实现（可被训练或加载权重），不应包含 I/O 与 HTTP 逻辑。  
配置/权重：外部文件，提供可配置性与快速替换模型的能力。

**数据流（重点字段/类型）**  
上传文件 → 临时文件（磁盘，任意格式） → WAV（临时转） → numpy 1D float array（librosa, shape: [n_samples]，sr=16000） → pad/cut -> torch.Tensor shape [1, N] -> 送入模型 -> model 返回 (last_hidden, logits) -> logits[0,1] 作为 “真实” 分数 -> 与阈值比较 -> label + score 返回给客户端。

**错误处理与健壮性建议（基于代码观察）**  
ffmpeg.exe 依赖：app.py 检测本地 ffmpeg，否则给出警告；在生产环境应明确依赖或容器化。  
临时文件清理：当前实现有 finally 清理，建议在异常情况下也记录临时文件路径以便排查。  
并发与性能：模型在初始化时加载到 `predictor` 实例（单例），适合长期运行；但大并发场景下需注意 GPU 内存与推理队列，可加入任务队列或批量推理。  
日志：当前使用 print，建议替换为标准日志（logging）以便运维。
